# Standing decisions and constraints

Decisions that have been made and should not be re-litigated unless explicitly revisited. New analysis must respect these unless asked to challenge them.

## Format

Number each decision. Keep entries terse. Reference them by number from other docs.

```
N. **Short title.** One-paragraph rationale. Conditions for revisiting.
```

## Decisions

> Add entries here as decisions accumulate. Examples of what belongs:
>
> - Architectural choices ("we use X, not Y")
> - Sizing or threshold rules
> - Out-of-scope boundaries ("we don't do Z")
> - Vendor or tool selections
>
> What does NOT belong: ephemeral state, todo items, things still under debate.

## Things explicitly under consideration but not committed

> Use this section for proposals that are live but not yet decided. Move to "Decisions" once committed.

# Meeting Summary Safety Policy

The output should be safe for broad internal Dark Matter sharing. It should capture work value without preserving private context or avoidable interpersonal detail.

## Keep

- Decisions and rationale that affect the work.
- Action items, owners, and deadlines when supported by the source.
- Open questions and blockers.
- Product, engineering, operations, customer, market, and strategy context.
- Useful links, repo names, ticket IDs, project names, and public references.
- Neutral disagreement when it is necessary to understand a decision or tradeoff.

## Remove

- Personal life details, relationships, family matters, health details, finances, travel logistics, or anything unrelated to work.
- Gossip, rumors, venting, speculation about motives, or conversational color about teammates.
- Feedback about a named teammate unless it is already formalized as neutral work process, a decision, or an action item.
- Compensation, HR-sensitive details, recruiting-sensitive details, personnel decisions, or interpersonal conflict.
- Customer, user, candidate, or counterparty PII.
- Credentials, API keys, secrets, private URLs, wallet-private material, account IDs, phone numbers, email addresses, and physical addresses.
- Legal strategy, privileged material, investor-sensitive material, or incident-response details unless the user explicitly chooses an appropriate restricted destination.
- Direct quotes that preserve avoidable personal or sensitive context.

## Transform

- Convert conversational back-and-forth into neutral decisions, context, risks, and next steps.
- Convert critique of a person into a process concern only when that work concern is necessary and supported by the source.
- Convert long transcript passages into concise bullets.
- Replace sensitive identifiers with generic labels only when the work point remains useful.
- Drop details entirely when anonymization would still make the person or private situation obvious.

## Review Rules

- If a detail would surprise someone to see in a company-wide note, omit it.
- If a detail is true but unnecessary for the work record, omit it.
- If a source contains sensitive material mixed with useful work content, preserve the work content and report only the removed category.
- Do not list the sensitive details that were removed.
- If the destination should be restricted instead of broad internal, stop and ask the user where it belongs.

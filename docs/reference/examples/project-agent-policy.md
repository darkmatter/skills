# Reference example — project-local `.agent/policy/`

This shows the project-local policy overlay I would stamp into each downstream repository.

Recommended project structure:

```text
.agent/
  policy/
    README.md
    engineering-practices.md
    agent-practices.md
    review-gates.md
    project-exceptions.md
```

Example `.agent/policy/README.md`:

```md
# Project policy

This directory contains mandatory project-local policy for agents and humans working in this repository.

Read order:

1. `engineering-practices.md` — inherited darkmatter engineering practices
2. `agent-practices.md` — agent behavior rules
3. `review-gates.md` — when review is required
4. `project-exceptions.md` — approved deviations from inherited rules

## Inheritance

This project inherits policy from `darkmatter/skills`. If a project policy conflicts with org policy, the stricter rule wins unless `project-exceptions.md` records an approved exception.

## What belongs here

- Mandatory engineering practices
- Review requirements
- Exception records
- Risk controls
- Project-specific enforcement rules

## What does not belong here

- Current project facts; use `.agent/context/overview.md`
- Architecture decisions; use `.agent/context/decisions.md` or ADRs
- Recurring procedures; use `.agent/workflows/`
- Tutorials; use skills
```

Example `.agent/policy/project-exceptions.md`:

```md
# Project exceptions

Exceptions are approved deviations from inherited practices. If a practice is skipped and no exception applies, stop and ask.

## Format

Each exception must include:

- ID
- Practice waived
- Scope
- Rationale
- Approver
- Review or expiry date
- Replacement control, if any

## Active exceptions

### EX-001 — TDD waived for generated migration files

- Practice waived: tests before behavior changes
- Scope: `prisma/migrations/**`
- Rationale: files are generated by Prisma and reviewed via migration tests
- Approver: cooper
- Review date: 2026-08-01
- Replacement control: run `pnpm prisma migrate diff` and `pnpm test tests/db/migrations.test.ts`

### EX-002 — End-of-turn AI review optional for docs-only edits

- Practice waived: non-trivial diff review
- Scope: Markdown-only changes under `docs/**` that do not change operational runbooks
- Rationale: PR review is sufficient for narrative docs
- Approver: engineering
- Review date: 2026-08-01
- Replacement control: spellcheck and linkcheck in CI

## Expired exceptions

Move expired exceptions here with a note about whether they were renewed, removed, or replaced by permanent policy.
```
